Org Governance

组织级别规范

面向 AI 编码时代的统一治理框架

封面 01
Why

为什么需要组织级规范

AI 编码能力快速增强，但行为边界、输出稳定性和交付标准并不会自动统一。

规则口径不一致 团队之间标准漂移，导致交付质量波动
安全底线难统一 没有统一红线就会把风险下放到项目现场
流程容易被跳过 直接写代码会让规格、设计和验证缺位
治理不可审计 没有统一规范就难形成可检查、可追责的闭环
必要性 02
Principles

四条设计原则

组织层定义底线，不定义技术栈细节

保持通用约束，不把 Java、Vue、MyBatis 等实现细节塞进组织层。

技术栈层可以更严格，但不能更宽松

技术栈层负责把底线进一步具体化，但不得突破组织约束。

每条规则必须可被 AI 在特定环节执行

规则不是口号，必须能在生成、评审、发布等环节被落实。

无法执行的规则不进入正式约束

确保规则具备落地性，避免只有形式没有执行抓手。

体系地图 04
Core Lines

三条核心约束主线

AI 行为

AI 编码行为底线

  • 编码前必须思考，不假设、不隐藏困惑
  • 用最少的代码解决问题，不过度推测
  • 只碰必须碰的代码，匹配现有风格
  • 定义成功标准，测试先行，循环验证
安全

安全红线

  • 禁止硬编码密钥、密码、token
  • 禁止 SQL 拼接，必须参数化查询
  • 禁止明文存储密码，必须使用安全哈希机制
  • 禁止 eval、反序列化注入等危险操作
质量

代码质量规范

  • 所有错误必须明确上报，不得静默吞掉
  • 测试覆盖正常路径和异常路径
  • 代码审查不得走过场
  • 禁止占位实现、过度注释等 No-Go 反模式
核心底线 05
Pipeline

流程治理

proposal 明确为什么做、做什么
design 说明如何实现、影响范围与关键决策
specs 定义行为、输入输出、异常与测试场景
tasks 拆解执行步骤、依赖与验证要求
implementation 只有在 gate 满足后才进入编码
主线：proposal → design → specs → tasks → implementation

每个阶段有 gate（准入条件）和 owner（负责人），不是直接写代码，而是先把目标、规格、设计和任务拆解清楚。

研发过程 06
Artifacts

Artifact 规则

proposal

为什么做、做什么

  • 背景与目标
  • 变更内容与影响分析
  • 非目标边界
  • 不得写技术实现方案
specs

需求和行为定义

  • 前置条件与输入输出
  • 预期行为与异常场景
  • 测试场景定义
  • 必须可追溯到 proposal
design

如何实现

  • 需求引用与变更范围
  • 方案设计与数据保护
  • 错误处理方案
  • 不得过度设计、不得拆分任务
tasks

如何落地执行

  • 验收标准与优先级
  • 设计引用与前置依赖
  • 测试与验证要求
  • 每个 task 可独立完成和验证

每阶段有"不能写什么"的边界，有跨阶段追溯关系，specs 与 tasks 中测试要求贯穿始终。

协作边界 07
Governance

冲突与豁免机制

默认冲突策略

stricter wins：更严格者优先；数值取更大、范围取更窄、严重级别取更高。

不可豁免范围

SEC-* 和 NG-* 不可豁免，安全红线和 No-Go 反模式是组织底线。

可审批豁免

AI-*、PL-*、CQ-*、ORG-* 可申请，但需要说明风险、截止时间与补救计划。

AI 的裁定边界

遇到无法自动判断的定性冲突，AI 必须停下来询问人工，不得自行拍板。

治理裁定 08
Layer Split

组织层 vs 技术栈层

组织层

定义底线

  • 原则与规则声明
  • 严重级别与执行者
  • 通用 detection_principle
  • 冲突与豁免治理逻辑
技术栈层

定义具体检测

  • patterns / AST / lint 规则
  • false positive 说明
  • true positive 示例
  • 语言与框架专项约束

组织层定义底线，不定义技术栈细节；技术栈层可以更严格，但不能更宽松。各技术栈在组织层底线上进一步具体化检测规则。

职责分层 09
Closing

总结与推行建议

组织级规范不是为了增加文档负担，而是为了让 AI 时代的软件交付更可控、更一致、更可审计。

1

统一组织层底线

先让所有团队对安全红线、质量底线和流程门禁形成共识

2

补齐技术栈层规则

各技术栈在底线之上定义具体的检测模式和框架专项约束

3

形成执行闭环

接入 AI 自动检测、CI/CD 平台门禁和人工审批，让规则真正落地
